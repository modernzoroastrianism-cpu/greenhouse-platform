# AMNI Outcome-Based Agent Architecture

## Core Philosophy

**AMNI agents think in OUTCOMES, not TASKS.**

Instead of:
- âŒ "Task: Post to feed 3x per day"
- âŒ "Task: Check sensors every hour"
- âŒ "Task: Send 10 outreach messages"

Agents think:
- âœ… "Outcome: Achieve $10K monthly revenue"
- âœ… "Outcome: Keep all plants healthy and producing"
- âœ… "Outcome: Build a team of 25 active members"

The agent figures out WHAT to do to achieve the outcome. The human defines SUCCESS, not the steps.

---

## Why Outcome-Based?

### Task-Based Problems
- Agents do busywork even when it doesn't help
- No adaptability when circumstances change
- Success = "I did the thing" not "I achieved the result"
- Human has to micromanage the HOW

### Outcome-Based Benefits
- Agents are creative problem-solvers
- Adapts strategy when something isn't working
- Success = measurable results
- Human focuses on WHAT matters, agent figures out HOW

## Implementation

### Outcome Schema
```typescript
interface Outcome {
  id: string
  name: string
  type: 'growth' | 'quality' | 'efficiency' | 'network'
  metric: string
  unit: string
  target: number
  current: number
  deadline?: Date
  priority: number
  assignedAgents: string[]
  subOutcomes?: Outcome[]
  strategies?: Strategy[]
  progress: ProgressEntry[]
}

interface Strategy {
  id: string
  description: string
  estimatedImpact: number
  resources: Resource[]
  status: 'proposed' | 'approved' | 'active' | 'completed' | 'abandoned'
  results?: StrategyResult
}

interface ProgressEntry {
  timestamp: Date
  value: number
  note: string
  agentId: string
}
```

### Agent Decision Loop
```
while (outcome not achieved):
    1. Assess current state vs target
    2. Identify gap
    3. Generate/evaluate strategies
    4. Select best strategy given constraints
    5. Execute strategy
    6. Measure impact
    7. If working â†’ continue
       If not working â†’ try different strategy
    8. Report progress to human
```

## Key Principles

1. **Humans define WHAT, agents figure out HOW**
2. **Measure outcomes, not activities**
3. **Agents adapt strategies based on results**
4. **Progress reports focus on outcome metrics**
5. **Self-correction within autonomy bounds**
6. **Escalate when strategy isn't working**
